class Expression
#include <expression.h>

Expression that consists of mutiple steps and can be evaulated by substituting variable values.

Expression can include:

  • Typed constants.

  • Untyped variables.

  • Arithmetic, logical or bitwise operations.

  • Conditional branches.

  • Extension functions.

Result type is defined by types of constants and variables. Operations, conditions and functions may enforce typing rules on their arguments. If at some point types become incompatible, the error is returned.

Evaluation state (variable substitutions, evaluation results) are stored in Context instance, so a single expression can be evaluated using different contexts and produce different results.

tparam Store

type that can store calculation results.

tparam Funs

pack of extension function types.

struct null
#include <null.h>

Type of null literal used inside pmql to indicate an absense of value. All standard operations with null literals are well-defined:

  • Arithmetic and bitwise operations: if any argument is null, then the result is null.

  • Comparison: null is equal only to itself and always less than any non-null value.

Public Functions

inline operator bool() const

Nulls are implicitly convertible to bool(false) to be usable as conditions.
